how upper works, adding ascii value and modulus of 26?

No, .upper() does not work by adding ASCII and taking modulus 26. That idea comes from Caesar cipher / alphabet shifting, not case conversion.

How upper() actually works
In Python, str.upper() uses Unicode case mapping tables, not arithmetic.

Example:

'a'.upper()  # 'A'
'b'.upper()  # 'B'
Internally Python checks a lookup table that says:

'a' → 'A'
'b' → 'B'
'c' → 'C'
...
For ASCII letters it could be implemented with a simple offset, because:

'a' = 97
'A' = 65
difference = 32
So conceptually:

uppercase = lowercase - 32
Example:

ord('a') = 97
97 - 32 = 65
chr(65) = 'A'
But Python does not rely on this trick because it must support Unicode characters.

Example:

'ß'.upper()
Output:

'SS'
This cannot be done with ASCII arithmetic.

Summary
Method	Used by Python?	Reason
ASCII arithmetic (-32)	❌	Only works for English letters
Modulus 26	❌	Used for ciphers
Unicode mapping tables	✅	Works for all languages
If you wanted to implement ASCII-only uppercase manually
def ascii_upper(c):
    if 'a' <= c <= 'z':
        return chr(ord(c) - 32)
    return c

Retry Context Manager
import time

class Retry:
    def __init__(self, attempts=3, delay=1):
        self.attempts = attempts
        self.delay = delay

    def __enter__(self):
        return self

    def run(self, func, *args, **kwargs):
        for i in range(self.attempts):
            try:
                return func(*args, **kwargs)
            except Exception as e:
                print(f"Attempt {i+1} failed:", e)
                if i < self.attempts - 1:
                    time.sleep(self.delay)
        raise Exception("All attempts failed")

    def __exit__(self, exc_type, exc, tb):
        return True

    result = r.run(unstable)
    print(result)
