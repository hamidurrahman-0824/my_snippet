ONE-TIME SETUP (PER REPOSITORY)
CREATE A REPO ON GITHUB

GO TO GITHUB → CLICK NEW REPOSITORY → GIVE IT A NAME (E.G., CLEAN-CODE-PRACTICE).

COPY THE REPO URL (HTTPS OR SSH).

INITIALIZE GIT LOCALLY

BASH
CD PATH/TO/YOUR/PROJECT
GIT INIT
GIT REMOTE ADD ORIGIN <YOUR-REPO-URL>
DAILY WORKFLOW (ADD, COMMIT, PUSH)
WHENEVER YOU’VE MADE CHANGES AND WANT TO UPLOAD THEM:

CHECK STATUS

BASH
GIT STATUS
THIS SHOWS WHICH FILES ARE MODIFIED OR NEW.

STAGE CHANGES

BASH
GIT ADD .
. MEANS “ADD ALL CHANGES IN THIS FOLDER.”

YOU CAN ALSO ADD SPECIFIC FILES: GIT ADD FILE.PY.

COMMIT CHANGES

BASH
GIT COMMIT -M "MEANINGFUL COMMIT MESSAGE"
EXAMPLE: "ADDED PRIMEITER CLASS WITH FIXES"

PUSH TO GITHUB

BASH
GIT PUSH ORIGIN MAIN
IF YOUR BRANCH IS CALLED MASTER, USE MASTER INSTEAD OF MAIN.

FIRST PUSH MAY REQUIRE:

BASH
GIT PUSH -U ORIGIN MAIN
(-U SETS UPSTREAM SO FUTURE PUSHES CAN JUST BE GIT PUSH).
